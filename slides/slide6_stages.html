Stages

Yet another picture..

changed/modified

  • changes we made to the code
  • can be easily reverted by "git checkout [filename]"

staged

  • work we want to keep (and not lose via git checkout)
  • code that is ready to commit (parts of a feature we work on)

commit

  • work that is done
  • code we want to save or keep for sure
  • versioning starts here (every commit has a hash)
  • creates a point to jump back (checkout) to again (via hash)
  • ready to be "pushed"
